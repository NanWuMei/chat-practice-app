# 🪞 镜子 — 情感聊天练习系统

> 你以为你在跟 AI 聊天，其实你在照镜子。

一个本地运行的中文聊天训练工具。与 AI 角色模拟微信聊天，通过苏格拉底式的自我提问发现自己的聊天模式——不评分、不给答案，只帮你看见自己。

## 设计理念

传统工具告诉你"这句话打8分"、"你应该这样说"。镜子不同：

- **不评分** — 没有 1-10 分，没有"打败 80% 用户"
- **不给改法** — 不会说"你应该改成 XXX"
- **只提问** — 苏格拉底式的开放式问题，帮你自己发现
- **看见自己** — 你的模式、你的节奏、你的盲区

## 核心流程

```
聊天 → 结束 → 自动复盘
              ↓
         关键时刻检测（纯算法，6种类型）
              ↓
         苏格拉底提问（仅1次AI调用）
              ↓
         你写下反思 → 行动锚点 → 存档
              ↓
         5次后解锁「模式发现」（按场景分类的历史回顾）
```

### 关键时刻类型

| 类型 | 含义 |
|------|------|
| KM_A | 她回复特别长（投入度上升） |
| KM_B | 她回复特别短（投入度下降） |
| KM_C | 你分享了自己的经历/感受 |
| KM_D | 你连续追问但没有分享 |
| KM_E | 她主动发起新话题 |
| KM_F | 对话突然中断 |

### 交互共振系统

关系状态由「交互共振分数」驱动，不绑定阶段：

- 共振分数 ±10 范围浮动，由 KM 统计自动计算
- 温度（T1-T5）由共振分数独立决定，不受阶段限制
- 阶段（试探期→升温期→暧昧期→突破期）基于长期趋势独立升降

### 行动锚点 & 回溯追踪

每次复盘结束时，你可以写下"下次想试的一个小变化"。下次聊天开始时，聊天界面顶部会显示这个锚点作为提醒。下次复盘时，系统会问你："上次你说想试试……试了吗？"

## 内置角色

### 练习对象

| 角色 | 来源 | 特点 |
|---|---|---|
| 梁友安 | 《爱情而已》 | 27岁体育经纪人，独立、有边界感、对套路免疫 |

梁友安基于电视剧角色深度蒸馏，包含 5 个核心心智模型、8 条决策启发式、完整的表达 DNA 和情绪依恋风格。她会根据「交互共振分数」动态调整回复风格——温度低时简短克制，温度高时自然展开。

### 复盘提问者

| 角色 | 身份 | 作用 |
|---|---|---|
| 苏格拉底 | 哲学家 | 复盘时通过开放式提问帮你自我审视 |

苏格拉底基于柏拉图全部对话录、色诺芬回忆录等 2000 年研究传统深度蒸馏，提炼 6 个核心心智模型、8 条决策启发式。他不评判、不给建议，只通过追问和反讽引导你自己发现矛盾。每次复盘只调用 1 次 AI，生成针对关键时刻的现象学问题。

## 快速开始

### 1. 克隆项目

```bash
git clone https://github.com/NanWuMei/chat-practice-app.git
cd chat-practice-app
```

### 2. 安装依赖

```bash
npm install
```

### 3. 配置 API Key

```bash
cp .env.example .env
```

编辑 `.env`，填入你的 API Key：

```
MIMO_API_KEY=你的API密钥
```

### 4. 启动

```bash
npm run dev:all
```

- 前端：http://localhost:5173
- 后端：http://localhost:8787

## 常用命令

```bash
npm run dev:all    # 同时启动前端和后端
npm run dev        # 仅前端
npm run server     # 仅后端
npm run test       # 运行测试
npm run build      # 构建生产版本
```

## 技术栈

| 层 | 技术 |
|---|---|
| 前端 | React 19 + TypeScript + Vite |
| 后端 | Express + TypeScript |

## 项目结构

```
src/
├── client/          # 前端 React 应用
│   ├── components/  # ChatScreen / ReviewScreen / ...
│   ├── pages/       # 路由页面
│   └── store/       # Zustand 状态管理
├── server/          # 后端 Express 服务
│   ├── ai/          # 检测算法 + 提示词 + AI 调用

│   └── data.ts      # 角色数据
├── shared/          # 前后端共享类型 + 校验器
└── skills/          # 蒸馏后的角色 Skill
│   ├── liang-youan/ # 梁友安角色操作系统
│   └── socrates-perspective/ # 苏格拉底思维操作系统
tests/               # Vitest 测试
```
